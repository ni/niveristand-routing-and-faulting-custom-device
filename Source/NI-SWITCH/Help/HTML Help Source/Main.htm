NI-SWITCH

The NI-SWITCH Custom Device supports NI-SWITCH routing modules by enumerating all available endpoints, and making them available to the Routing and Faulting Custom Device.

Configuring the Custom Device

Set Resource to the name of NI-SWITCH module in NI MAX.

Set Topology to the desired NI-SWITCH topology. The topology must correspond to the model number of the NI-SWITCH module. Some modules support multiple topologies.

NOTE: Configure the NI-SWITCH Custom Device before configuring the Routing and Faulting Custom Device.

Endpoint Configuration

Set endpoint configurations to modify the possible connections for the routing module. The following table lists the available endpoint configurations.

Configuration Description
Load Endpoint is unrestricted and can be connected to other load and source endpoints. This is the default configuration.
Source Endpoint is restricted and cannot be connected to another source endpoint.
Configuration Endpoint is reserved for internal hardware routing. This configuration cannot be used directly.

Normally Connected Endpoints

Some NI-SWITCH modules, such as the PXI-2510, contain normally connected endpoints. These endpoints are automatically connected by the NI-SWITCH driver when the device is initialized or reset.

If a pair of normally connected endpoints are referenced by a state in the Routing and Faulting custom device, these endpoints will be automatically managed by the custom device. If the pair is not referenced by any of the routing channel states, the endpoints will not be managed by the Routing and Faulting Custom Device. Instead, the endpoints will remain connected during deployment.
